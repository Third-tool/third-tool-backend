# 도메인 모델링 문서 작성 규칙

> 도메인별 모델링 문서(`card.md`, `deck.md`, `review.md`, `learning-facade.md`, `user.md`, `user-schedule.md`)를 작성·유지할 때 따라야 할 형식과 컨벤션입니다.
> 새 도메인을 추가하거나 기존 도메인 문서를 수정할 때 이 파일을 먼저 확인하세요.

---

## 1. 파일 분리 원칙

- BC(Bounded Context) **하나당 파일 하나**를 둡니다. 다중 BC를 한 파일에 묶지 않습니다.
- 파일명은 BC 디렉토리 이름의 kebab-case (`Card/` → `card.md`, `LearningFacade/` → `learning-facade.md`).
- 파일 위치: `docs/domain/{bc-name}.md`.
- 같은 도메인의 DB 스키마는 `docs/database/{bc-name}.md`에 별도로 둡니다 (도메인 모델 ↔ 테이블 정의 1:1 매칭).

## 2. 표준 섹션 구조

각 도메인 문서는 다음 순서를 지킵니다.

```markdown
# {BC명} 도메인 모델

> 한 줄 요약 — 이 BC가 표현하는 핵심 개념.

---

## 구성 요소

| 구분 | 클래스 | 역할 |
|---|---|---|
| Aggregate Root | ... | ... |
| Entity | ... | ... |
| Value Object | ... | ... |
| Domain Service | ... | ... |
| Enum | ... | ... |

---

## {클래스명} ({Aggregate Root | Entity | VO | Domain Service | Enum})

> 한 줄 정의.

### 설명
자유 서술. 도메인 의미와 의도.

### 속성
| 속성 | 타입 | 설명 |
|---|---|---|

### 행위
| 행위 | 시그니처 | 설명 |
|---|---|---|

### 규칙
- 불변식, 비즈니스 룰을 글머리 기호로.

### 검증
- 생성 시점, 수정 시점에서 거부되는 입력을 명시.

---

## 책임 분리

> 각 클래스가 가진 단일 책임을 1-3줄로 요약.
```

- 모든 도메인 문서가 이 구조를 따라야 검색·비교가 쉬워집니다.
- 섹션이 비어 있어도 헤더는 두고 `_TBD_`로 표시합니다 (구조 일관성 유지).

## 3. 클래스 분류 규칙

- **Aggregate Root** — 트랜잭션 경계를 책임지는 엔티티. BC당 보통 1-2개.
- **Entity** — 식별자가 있고 Aggregate Root에 종속되는 객체.
- **Value Object (VO)** — 식별자 없이 값으로 구분되는 객체. `@Embeddable` 또는 record.
- **Domain Service** — Aggregate에 자연스럽게 들어가지 않는 도메인 규칙. `model/` 패키지에 둡니다.
- **Enum** — 상태/유형 분류값.

## 4. 표 작성 규칙

- 속성 표는 `속성 | 타입 | 설명` 3열.
- 행위 표는 `행위 | 시그니처 | 설명` 3열. 시그니처 칼럼이 비대해지면 시그니처 칼럼을 빼고 본문에 코드 블록을 둡니다.
- Enum 값 표는 `값 | 설명` 2열.

## 5. 글쓰기 톤

- "~~해야 한다", "~~될 수 있다" 같은 모호한 표현 대신 `~한다`, `~생성할 수 없다` 같이 단정적으로 적습니다.
- 비즈니스 의도(예: "태그 3개 제한은 연결의 밀도를 높이기 위한 의도적 설계다")를 함께 서술해 단순 기술 제약과 구분합니다.
- 외부 시스템 의존(JPA·Spring 어노테이션)은 도메인 문서에서 다루지 않습니다 (DB 측은 `docs/database/`).

## 6. 변경 추적

- 도메인 모델 변경 시: 코드 → 본 문서 → CLAUDE.md / PROJECT_OVERVIEW.md 순으로 갱신합니다.
- 큰 변경(메서드 명 변경, 상태 추가)은 문서 하단에 "변경 요약" 표를 둘 수 있습니다 (`docs/domain/도메인모델.md`의 ReviewStep 섹션 참조).

## 7. 기존 모놀리식 문서

- 분리 이전의 통합 문서는 [`도메인모델.md`](도메인모델.md)에 보존되어 있습니다. 신규 작성된 도메인별 파일이 기존 내용을 모두 흡수할 때까지는 두 문서를 병행 유지합니다.
- 도메인별 파일에 내용을 옮긴 부분은 `도메인모델.md`에서 "→ [card.md](card.md) 로 이전" 같은 안내 줄로 대체할 수 있습니다.

## 8. 용어 정합성

- 도메인 용어는 [`용어사전.md`](용어사전.md) (Notion)에 정의된 표기를 따릅니다. 표준 표기와 다른 신규 용어를 만들 경우 사전을 먼저 갱신하세요.
