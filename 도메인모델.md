# 도메인 모델 요약
---

## Card
회상 가능한 학습 단위를 모델링한다.

### 설명
Card는 단순 메모 저장소가 아니다.
학습자가 맥락을 보고, 단서를 통해 떠올리고, 마지막에 자기 언어로 핵심을 압축하도록 돕는 학습 단위다.

Card는 MainNote, KeywordCue, Summary를 반드시 구분해서 가진다.

### 속성
- `id` : 카드 식별자
- `mainNote` : 학습 맥락 영역
- `summary` : 핵심 압축 영역
- `keywordCues` : 회상 단서 목록

### 행위
- `create(MainNote, Summary, List<String>)` : 카드 생성
- `changeMainNote(String textContent, String imageUrl)` : Main 수정
- `changeSummary(String value)` : Summary 수정
- `replaceKeywords(List<String> values)` : Keywords 전체 교체
- `addKeyword(String value)` : 단서 추가
- `removeKeyword(Long keywordCueId)` : 단서 제거

### 규칙
- 카드는 Main / Keywords / Summary를 분리해서 가진다.
- Main은 텍스트 또는 이미지 중 최소 하나를 포함해야 한다.(여러개도 가능하다.)
- Keywords는 최소 1개 이상이어야 한다.
- Summary는 비어 있으면 안 된다.
- Summary는 1~3문장 범위를 따라야 한다.
- 카드의 세 영역은 하나의 통합 텍스트로 저장하지 않는다.

### 검증
- `mainNote`가 `null`이면 안 된다.
- `summary`가 `null`이면 안 된다.
- `keywordCues`가 `null`이면 안 된다.
- 카드 생성 시 Keyword 목록은 최소 1개 이상이어야 한다.
- `replaceKeywords(...)` 호출 시에도 빈 목록으로 교체할 수 없다.
- `removeKeyword(...)` 수행 후 Keyword가 0개가 되면 안 된다.
- Aggregate는 “회상 가능한 최소 구조”가 깨지지 않도록 생성/수정 시점마다 일관성을 검사해야 한다.

---

## MainNote
학습의 맥락과 근거를 담는 영역을 모델링한다.

### 설명
MainNote는 정답 자체를 그대로 적는 공간이 아니다.
학습 장면, 설명, 도식, 이미지, 배경 근거를 담는 역할을 한다.

### 속성
- `textContent` : 본문 텍스트
- `imageUrl` : 이미지 경로
- `contentType` : Main 구성 타입(TEXT_ONLY / IMAGE_ONLY / MIXED)

### 행위
- `of(String textContent, String imageUrl)` : MainNote 생성

### 규칙
- textContent와 imageUrl이 둘 다 비어 있으면 안 된다.
- 텍스트만 있을 수 있다.
- 이미지만 있을 수 있다.
- 텍스트와 이미지가 함께 있을 수 있다.
- 구성 상태에 따라 contentType이 결정된다.

### 검증
- `textContent`, `imageUrl`는 저장 전에 trim 처리한다.
- `textContent`와 `imageUrl`이 모두 `null` 또는 blank이면 생성할 수 없다.
- 텍스트만 존재하면 `TEXT_ONLY`여야 한다.
- 이미지만 존재하면 `IMAGE_ONLY`여야 한다.
- 둘 다 존재하면 `MIXED`여야 한다.
- `contentType`은 외부에서 주입하지 않고 내부 규칙으로만 결정한다.

---

## KeywordCue
회상을 유도하는 단서를 모델링한다.

### 설명
KeywordCue는 사용자가 정답을 바로 보는 대신,
짧은 단서를 보고 스스로 떠올리게 만드는 인출 장치다.

### 속성
- `id` : 단서 식별자
- `value` : 단서 내용
- `card` : 소속 카드

### 행위
- `create(Card, String value)` : 단서 생성

### 규칙
- Keyword는 비어 있으면 안 된다.
- 하나의 카드는 여러 개의 Keyword를 가질 수 있다.
- 카드 생성 시 최소 1개 이상 필요하다.
- Keyword는 카드에 종속된다.

### 검증
- `value`는 trim 처리 후 저장한다.
- `value`가 `null` 또는 blank이면 생성할 수 없다.
- `card`가 `null`이면 생성할 수 없다.
- 공백만 있는 문자열은 유효한 단서로 인정하지 않는다.
- 단건 Keyword는 자신의 값 유효성만 검사하고, 최소 개수 검사는 Card Aggregate가 담당한다.
---

## Summary
핵심을 자기 언어로 짧게 압축한 결과를 모델링한다.

### 설명
Summary는 원문 복붙이 아니라,
학습자가 이해한 핵심을 1~3문장으로 압축한 결과다.

### 속성
- `value` : 요약 내용

### 행위
- `of(String value)` : Summary 생성

### 규칙
- Summary는 비어 있으면 안 된다.
- Summary는 1~3문장 범위여야 한다.
- Summary는 너무 긴 설명문이 아니라 짧은 압축이어야 한다.
- 카드당 Summary는 하나만 가진다.

### 검증
- `value`는 trim 처리 후 저장한다.
- `value`가 `null` 또는 blank이면 생성할 수 없다.
- 문장 수는 1~3문장 범위여야 한다.
- 문장 수 판정 시 빈 문장은 제외한다.
- Summary는 단순 존재 여부만이 아니라 “짧은 압축”이라는 형태 규칙까지 함께 검사한다.

---

## MainContentType (enum)
MainNote의 구성 방식을 모델링한다.

### 값
- `TEXT_ONLY` : 텍스트만 존재
- `IMAGE_ONLY` : 이미지만 존재
- `MIXED` : 텍스트와 이미지가 함께 존재

### 규칙
- MainNote 생성 시 text/image 존재 여부에 따라 자동 결정된다.

### 검증
- enum 값은 외부 입력값으로 직접 결정하지 않는다.
- MainNote 내부 상태를 기준으로만 계산된다.
- 잘못된 조합 상태와 enum 값이 따로 놀지 않도록 항상 MainNote 생성 로직 안에서 함께 결정한다.

# 책임 분리

---

## Card Aggregate
### 책임
- Main / Keywords / Summary를 하나의 카드로 묶어 관리한다.
- 카드가 “회상 가능한 학습 단위” 구조를 유지하도록 전체 일관성을 보장한다.
- Main, Keywords, Summary가 서로 섞이지 않고 역할별로 분리되도록 관리한다.
- 카드 생성 시 최소한의 완전한 학습 구조를 갖추도록 강제한다.
- Keywords 목록의 교체, 추가, 제거 같은 변화 흐름을 통제한다.
- 카드가 최소 1개 이상의 Keyword를 유지하도록 보장한다.
- 회상 흐름의 중심 단위로서 내부 구성요소들을 조합하고 유지한다.

결론: **애그리게이트 책임**

---

## MainNote
### 책임
- 학습의 본문 맥락과 근거를 표현한다.
- 텍스트/이미지 기반 학습 재료를 관리한다.
- 텍스트와 이미지가 모두 비어 있는 잘못된 Main 생성을 막는다.
- Main의 구성 상태를 기준으로 content type을 결정한다.

결론: **학습 맥락 책임**

---

## KeywordCue
### 책임
- 회상을 유도하는 단일 단서를 표현한다.
- 공백 또는 무의미한 단서가 저장되지 않도록 막는다.
- 카드에 종속된 단서로서 자신의 값 일관성을 유지한다.

결론: **회상 단서 책임**

---

## Summary
### 책임
- 학습 내용을 짧게 압축한 결과를 표현한다.
- 비어 있는 요약을 막는다.
- 1~3문장 범위를 벗어나는 요약을 막는다.
- 긴 설명문이 아니라 핵심 압축 결과라는 성격을 유지한다.

결론: **핵심 압축 책임**
