## 1. decks

덱(Deck)의 메타 정보, 계층 구조, 공개 라이브러리 여부를 저장하는 테이블입니다.

| No. | 컬럼 한글명 | 컬럼 영문명 | 데이터 타입 | 제약 조건 | 기본값 | 비고 |
|-----|------------|------------|------------|----------|--------|------|
| 1 | 덱 ID | id | BIGINT | PK, AUTO_INCREMENT | — | 덱의 고유 식별자 |
| 2 | 덱 이름 | name | VARCHAR(100) | NOT NULL | — | 동일 유저 내 이름 중복 불가 (uk_deck_user_name) |
| 3 | 유저 ID | user_id | BIGINT | FK, NOT NULL | — | users.id 참조 |
| 4 | 부모 덱 ID | parent_deck_id | BIGINT | FK, NULL | NULL | decks.id 자기 참조. NULL이면 루트 덱 |
| 5 | 계층 깊이 | depth | INT | NOT NULL | 0 | 루트=0, 자식=부모depth+1 자동 계산 |
| 6 | 최근 접근 시각 | last_accessed | DATETIME | NULL | — | 덱 접근 시마다 갱신 |
| 7 | 공개 라이브러리 여부 | on_library | BOOLEAN | NOT NULL | false | true이면 공개 라이브러리에 노출 |
| 8 | 공개 시각 | published_at | DATETIME | NULL | NULL | on_library=true 시 정렬 기준으로 사용 |

**인덱스 / 제약조건**

| 종류 | 이름 | 대상 컬럼 | 비고 |
|------|------|----------|------|
| UNIQUE | uk_deck_user_name | (user_id, name) | 동일 유저 내 덱 이름 중복 방지 |
| FK | fk_deck_user | user_id → users.id | |
| FK | fk_deck_parent | parent_deck_id → decks.id | |

## 2. cards

카드의 기본 정보와 Main / Summary 영역을 저장하는 테이블입니다.

| No. | 컬럼 한글명 | 컬럼 영문명 | 데이터 타입 | 제약 조건 | 기본값 | 비고 |
|-----|------------|------------|------------|----------|--------|------|
| 1 | 카드 ID | id | BIGINT | PK, AUTO_INCREMENT | — | 카드의 고유 식별자 |
| 2 | 덱 ID | deck_id | BIGINT | FK, NOT NULL | — | decks.id 참조 |
| 3 | 메인 텍스트 | main_text_content | TEXT | NULL | NULL | 학습의 본문 맥락 텍스트. ※ 코드 기준 TEXT — 명세와 불일치 |
| 4 | 메인 이미지 경로 | main_image_url | VARCHAR(2048) | NULL | NULL | 학습의 본문 이미지 URL. ※ 코드 기준 2048 — 명세와 불일치 |
| 5 | 메인 구성 타입 | main_content_type | VARCHAR(20) | NOT NULL | — | `TEXT_ONLY` / `IMAGE_ONLY` / `MIXED` 중 하나 |
| 6 | 요약 내용 | summary_value | TEXT | NOT NULL | — | 1~3문장 범위의 핵심 압축 내용. ※ 코드 기준 column명 `summary_value` — 명세와 불일치 |
| 7 | 삭제 여부 | deleted | BOOLEAN | NOT NULL | false | Soft Delete. true이면 논리 삭제 상태 |
| 8 | 생성 일시 | created_date | DATETIME | NOT NULL, 수정 불가 | — | @CreationTimestamp 자동 설정 |
| 9 | 수정 일시 | updated_date | DATETIME | NULL | — | @UpdateTimestamp 자동 갱신 |

**인덱스 / 제약조건**

| 종류 | 이름 | 대상 컬럼 | 비고 |
|------|------|----------|------|
| FK | fk_card_deck | deck_id → decks.id | |
| INDEX | idx_card_deleted | deleted | Soft Delete 필터 조회 성능 |

## 3. keyword_cues

카드에 연결된 회상 단서를 저장하는 테이블입니다.  
Card Aggregate Root 하위의 자식 엔티티로, Card 삭제 시 CascadeType.ALL + orphanRemoval로 함께 삭제됩니다.

| No. | 컬럼 한글명 | 컬럼 영문명 | 데이터 타입 | 제약 조건 | 기본값 | 비고 |
|-----|------------|------------|------------|----------|--------|------|
| 1 | 단서 ID | keyword_cue_id | BIGINT | PK, AUTO_INCREMENT | — | 회상 단서의 고유 식별자. ※ 코드 기준 column명 `keyword_cue_id` |
| 2 | 카드 ID | card_id | BIGINT | FK, NOT NULL | — | cards.id 참조 |
| 3 | 단서 내용 | value | VARCHAR(200) | NOT NULL | — | 공백 불가, trim 처리 후 저장. ※ 코드 기준 `value` VARCHAR(200) — 명세와 불일치 |
| 4 | 단서 순서 | cue_order | INT | NULL | NULL | **⚠️ 코드에 미구현.** 표시 순서 관리 시 별도 구현 필요 |

**인덱스 / 제약조건**

| 종류 | 이름 | 대상 컬럼 | 비고 |
|------|------|----------|------|
| FK | fk_keyword_cue_card | card_id → cards.id | Cascade 삭제 대상 |
| INDEX | idx_keyword_cue_card | card_id | 카드별 단서 조회 성능 |

## 테이블 관계 요약

```
users
  └── decks (user_id FK)
        └── decks (parent_deck_id FK, 자기 참조)
              └── cards (deck_id FK)
                    └── keyword_cues (card_id FK, CASCADE ALL)
```
