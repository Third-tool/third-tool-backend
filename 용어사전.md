| 용어 | 정의 | 주요 속성 | 관계 | 주요 역할 |
|---|---|---|---|---|
| **Card** | 회상 가능한 학습 단위. 학습 맥락, 인출 단서, 핵심 압축을 하나로 묶는 카드 애그리게이트 | `id`, `mainNote`, `summary`, `keywordCues` | `MainNote`(임베디드), `Summary`(임베디드), `KeywordCue`(일대다) | 회상 구조 유지, 카드 전체 일관성 보장, 단서 목록 변경 통제 |
| **MainNote** | 학습의 본문 맥락과 근거를 담는 값 객체 | `textContent`, `imageUrl`, `contentType` | `Card`에 소속(임베디드), `MainContentType` 사용 | 학습 장면/설명/이미지 보관, Main 구성 상태 판단 |
| **KeywordCue** | 사용자가 스스로 떠올리도록 돕는 단일 인출 단서 | `id`, `value` | `Card`(다대일) | 회상 단서 표현, 공백/무의미한 값 방지 |
| **Summary** | 학습 내용을 자기 언어로 1~3문장 안에 압축한 값 객체 | `value` | `Card`에 소속(임베디드) | 핵심 압축, 빠른 재인출 지원, 요약 규칙 보장 |
| **MainContentType** | MainNote의 구성 상태를 나타내는 열거형 | `TEXT_ONLY`, `IMAGE_ONLY`, `MIXED` | `MainNote` 내부 상태에 의해 결정 | 텍스트/이미지 조합 상태 표현 |
| **CardRepository** | Card 애그리게이트를 저장/조회하는 리포지토리 | - | `Card` 영속화 담당 | 카드 저장, 조회, 영속성 추상화 |
| **CardCreateRequest** | 외부에서 카드 생성 입력을 전달하는 요청 DTO | `mainText`, `mainImageUrl`, `keywords`, `summary` | `CardCommandService` 입력값 | 외부 입력 수집, 1차 형식 검증 |
| **CardResponse** | 카드 생성/조회 결과를 외부에 반환하는 응답 DTO | `id`, `mainText`, `mainImageUrl`, `mainContentType`, `keywords`, `summary` | `CardMapper`를 통해 생성 | API 응답 표현 |
| **CardMapper** | 도메인 객체를 응답 DTO로 변환하는 매퍼 | - | `Card` → `CardResponse` | 표현 계층과 도메인 계층 분리 |
| **CardCommandService** | 카드 생성 유스케이스를 실행하는 애플리케이션 서비스 | - | `CardRepository`, `CardMapper`, `CardCreateRequest` 사용 | 카드 생성 흐름 조립, 도메인 객체 저장, 응답 반환 |
| **CardController** | 카드 API 요청을 처리하는 어댑터 계층 | - | `CardCommandService` 호출 | HTTP 요청/응답 처리, 카드 생성 API 제공 |
| **Card Aggregate** | Card를 중심으로 MainNote, KeywordCue, Summary를 묶는 일관성 경계 | `mainNote`, `summary`, `keywordCues` | `Card`가 루트, 내부 구성요소 포함 | 회상 가능한 최소 구조 보장, 내부 변경 통제 |
